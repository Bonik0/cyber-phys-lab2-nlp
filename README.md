## Установка и запуск

### Автоматическая подготовка Ollama

Скрипт ставит Ollama (если её нет), при необходимости запускает **`ollama serve`** в фоне, выполняет **`ollama pull qwen2.5:0.5b`** и проверяет **`/api/tags`**.

```bash
chmod 777 setup_ollama.sh
./setup_ollama.sh
```

### Зависимости Python

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Запуск инференса

```bash
python3 run_inference.py
```

#### Параметры командной строки

| Параметр | По умолчанию | Назначение |
|----------|--------------|------------|
| `--base-url` | `http://127.0.0.1:11434` | корень URL Ollama |
| `--model` | `qwen2.5:0.5b` | имя модели |
| `--timeout` | `200` | таймаут одного запроса (сек) |
| `--md-out` | `report.md` | путь к Markdown-отчёту |
| `--prompts-file` | нет | файл с промптами (по строке на запрос) |

## Отчёт инференса и выводы

Фактические ответы модели и тайминги зависят от железа (CPU/GPU) и версии Ollama.

По уже полученному прогону можно отметить:

1. **Длительность** заметно различается по запросам (от нескольких секунд до десятков), что типично для локального инференса малой модели на CPU: длиннее ответ — дольше генерация.
2. **Качество** у **0.5B** неравномерное: на части тем ответы разумные и структурированные, на других возможны отказы («не могу…»), шаблонность или фактические неточности (ожидаемо для очень компактной модели без дообучения под задачу).
